Create TG-84 as a sub-workflow following the same pattern as TG-83 but for invoices. Triggered by TG-05 on invoice_sent.

Flow:
1. Trigger -> Enroll invoice (PATCH invoices set collections_status='enrolled')
2. Wait 3 days
3. Check status -> IF unpaid: Check SMS consent -> Send friendly SMS reminder -> PATCH collections_status='day3_sent'
4. Wait 7 days
5. Check status -> IF unpaid: Send email reminder -> PATCH collections_status='day10_sent'
6. Wait 7 days
7. Check status -> IF unpaid: Check SMS consent -> Send final SMS (firm tone) -> PATCH collections_status='escalated'

No-consent branches at Touch 1 and Touch 3 should connect to an exit node, NOT advance the sequence status.
