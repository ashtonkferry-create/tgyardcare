  <action>
Create TG-83 as a sub-workflow. This is the most complex workflow -- it uses Wait nodes to pace 3 touches over 14 days.

Flow:
1. Trigger (from TG-05) -> Enroll quote in follow-up (PATCH estimates set followup_status='enrolled')
2. Wait 48 hours
3. Check status (GET estimates) -> IF still pending: Check SMS consent -> IF consent: Send SMS Touch 1 -> Log -> PATCH followup_status='day2_sent'. IF no consent: go to Exit node (do NOT advance to Mark Day 2 Sent)
4. Wait 5 days
5. Check status -> IF still pending: Send Email Touch 2 (social proof via Brevo -- Brevo's own list suppression handles email consent) -> PATCH followup_status='day7_sent'
6. Wait 7 days
7. Check status -> IF still pending: Check SMS consent -> Send Final SMS Touch 3 -> PATCH followup_status='expired'

CRITICAL: After EVERY Wait node, re-query the estimates table to check if the customer already responded. If followup_status is NOT the expected value (e.g., it changed to 'accepted' or 'declined'), EXIT the sequence immediately.

CRITICAL: The no-consent branch of "Has Consent T1?" must connect to "Exit - Customer Responded" (or a dedicated no-consent exit node), NOT to "Mark Day 2 Sent". Sending to "Mark Day 2 Sent" without sending Touch 1 would skip the SMS but still advance the sequence, causing Touch 2 Email to be sent to a customer who never received Touch 1 and never consented.

Similarly, the no-consent branch of "Has Consent T3?" must connect to an exit node, NOT to "Mark Expired".
